현재 지도 화면에서는 두 가지 주요 문제가 확인된다.

지도 위에서 드래그로 이동이 불가능해 사용자가 화면을 자유롭게 탐색할 수 없음.

지도 진입 시 하단 메뉴바(탭바)가 사라지는 현상이 발생함.

또한, 초기 로딩 시 지도 중심이 서울 등 다른 지역으로 잡혀 있어
제주도 지역만 표시되도록 초기값을 지정할 필요가 있다.

아래와 같이 수정해 주세요.

1️⃣ 지도 드래그 이동 기능 활성화

사용자가 마우스(웹) 또는 손가락(모바일)으로 지도를 자유롭게 이동(Drag Pan)할 수 있도록 한다.

확대·축소 기능은 기존의 회색 3버튼(내 위치, +, −) 만 유지하고,
드래그 이동은 네이버 지도 기본 제스처로 활성화한다.

구현 예시

const map = new naver.maps.Map('map', {
  center: new naver.maps.LatLng(33.4996, 126.5312), // 제주도 중심 좌표
  zoom: 11,                  // 제주 전역이 보이도록 기본 확대값 조정
  draggable: true,           // ← 반드시 true (드래그 이동 허용)
  pinchZoom: true,           // 모바일 핀치 확대 지원
  scrollWheel: true,         // 마우스 휠 확대 허용
  keyboardShortcuts: false,  // 불필요한 단축키 비활성화
  disableDoubleTapZoom: false,
});


참고

center 옵션으로 초기 중심을 제주도 좌표(33.4996, 126.5312) 로 설정하면
앱 실행 시 바로 제주 지역이 보이게 된다.

필요 시 map.setOptions({ minZoom: 10, maxZoom: 18 }); 을 추가해
줌 범위를 제주도만 보이도록 제한할 수 있다.

바텀시트나 UI 오버레이가 지도 위를 덮는 경우에는
pointer-events: none 을 필요한 부분에만 지정하여
지도 제스처 이벤트가 정상적으로 전달되도록 한다.

모바일에서는 바텀시트 스크롤과 드래그 이벤트가 겹치지 않도록
리스트 스크롤이 끝났을 때만 지도 드래그를 활성화한다.

2️⃣ 메뉴바(탭바) 표시 복구 및 레이아웃 정리

지도 화면에서도 하단 탭바(Home, 탐색, 지도, 저장함, 내정보)가
항상 보이도록 유지해야 한다.

구조

<Layout>
  <Header />        <!-- 상단 검색바 -->
  <MapContainer />  <!-- 지도 및 바텀시트 -->
  <BottomNav />     <!-- 하단 메뉴바 -->
</Layout>


CSS 구조

.map-container {
  position: relative;
  height: calc(100vh - 64px); /* 하단 탭바 높이만큼 줄이기 */
  z-index: 900;
}

.bottom-nav {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  height: 64px;
  background: #fff;
  border-top: 1px solid #eee;
  z-index: 1000; /* 지도보다 위에 표시 */
}


지도와 바텀시트의 z-index는 900 이하,
하단 탭바는 1000 이상으로 고정해 항상 위에 노출되도록 한다.

바텀시트를 최대한 올려도 탭바를 덮지 않게 하고,
시각적으로 메뉴의 존재가 인식되도록 한다.

3️⃣ 초기 지도 범위 제주도 한정 설정

앱 첫 진입 시 지도의 표시 영역을 제주도 전체가 보이도록 제한한다.

네이버 지도 API에서는 bounds(지도의 가시 영역)를 지정할 수 있다.

예시 코드

// 제주도 경계 기준 좌표 범위
const jejuBounds = new naver.maps.LatLngBounds(
  new naver.maps.LatLng(33.1100, 126.1000), // 남서
  new naver.maps.LatLng(33.5800, 126.9800)  // 북동
);
map.fitBounds(jejuBounds); // 제주 전역 보이게 초기화


이후 지도를 드래그하거나 확대할 수 있지만,
map.setOptions({ minZoom: 10, maxZoom: 18 });로 제주 바깥으로 벗어나지 않게 제한한다.

이렇게 하면 초기 로딩 시 자동으로 제주 중심(제주시–서귀포 포함)만 표시된다.

4️⃣ QA 확인 포인트

 지도를 드래그로 자유롭게 이동할 수 있는지 확인

 확대·축소 버튼이 정상 동작하며 드래그 제스처와 충돌하지 않는지 확인

 앱 진입 시 중심이 항상 제주도로 설정되는지 확인

 하단 탭바가 지도 화면에서도 항상 표시되는지 확인

 바텀시트 확장 시 탭바가 덮이지 않고 유지되는지 확인

✅ 요약

지도는 드래그 이동 가능해야 하며,

초기 중심은 제주도로 고정하고 제주 지역만 보이게 설정한다.

하단 메뉴바는 항상 고정 표시되어야 하며
지도나 바텀시트가 이를 덮지 않도록 레이아웃과 z-index를 조정한다.
